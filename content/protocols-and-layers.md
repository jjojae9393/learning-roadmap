## 프로토콜의 의미와 종류

프로토콜은 컴퓨터들이 통신할 때 지켜야 하는 **약속이자 규칙**입니다. 역할에 따라 종류가 나뉩니다.

- **HTTP / HTTPS** — 웹 콘텐츠를 주고받는 프로토콜
- **TCP / UDP** — 데이터를 어떻게 전달할지 정하는 전송 프로토콜
- **IP** — 경로를 정하는 프로토콜
- **DNS** — 주소를 변환하는 프로토콜

여기서 짚어둘 중요한 점은, **HTTP/HTTPS는 암호화 프로토콜이 아니라 웹 프로토콜**이라는 것입니다.
`HTTPS = HTTP + TLS` 이며, 실제 암호화를 담당하는 건 별도의 보안 프로토콜인 **TLS**입니다.

## 계층 구조 (TCP/IP 4계층 모델)

통신 과정을 여러 층으로 나누고 각 층에 역할을 하나씩 맡긴 구조입니다.
윗층은 아랫층에 일을 맡기고, 아랫층의 내부 처리는 신경 쓰지 않습니다.

위에서부터 아래로:

- **애플리케이션 계층** — 무슨 내용을 주고받을지 정함 (HTTP, DNS, 이메일 등)
- **전송 계층** — 어떻게 전달할지 책임짐 (TCP, UDP)
- **인터넷 계층** — 어디로 갈지 경로를 정함 (IP)
- **네트워크 인터페이스 계층** — 실제 물리적 전송을 담당 (케이블, 와이파이 등)

데이터를 보낼 때는 위에서 아래로 내려가며 각 층이 정보를 덧붙이고,
받을 때는 아래에서 위로 올라가며 한 겹씩 벗겨 원래 내용을 복원합니다.

> 핵심은 HTTP, TCP, IP가 **경쟁 관계가 아니라 서로 다른 층에서 협력하는 관계**라는 점입니다.
> (OSI 7계층은 같은 개념을 더 세밀하게 나눈 모델입니다.)

## 암호화(TLS)의 위치

TLS는 전송 계층에서 실행되는 게 아니라, **전송 계층(TCP) 위, 애플리케이션 계층(HTTP) 아래**에 자리 잡고 동작합니다.
TCP가 깔아준 신뢰성 있는 통로 위에서 내용을 암호화합니다.

- 보낼 때: `HTTP → TLS(암호화) → TCP → IP`
- 받을 때: 그 역순

## 애플리케이션 계층의 정확한 의미

애플리케이션 계층은 '사용자에게 화면을 보여주는 단계'가 아니라,
**응용 프로그램끼리 무슨 내용을 주고받을지 정하는 규칙**입니다.

화면에 그려 보여주는 일은 브라우저 같은 프로그램이 하는 별개의 일이고,
애플리케이션 계층(HTTP)은 데이터를 주고받는 규칙까지를 담당합니다.
**데이터를 주고받는 규칙과 그것을 보여주는 방식이 분리돼 있는** 구조입니다.
